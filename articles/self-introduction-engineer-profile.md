# はじめまして。エンジニアの自己紹介

## 簡単なプロフィール

はじめまして。31歳のソフトウェアエンジニアです。

2017年に新卒でエンジニアとして入社し、気づけば約9年この仕事を続けています。
「作ったものが動く瞬間」が好きで、仕事も休日の個人開発もずっとコードを書き続けています。

---

## キャリアの変遷

### 最初の5年：組み込み開発

入社後まず配属されたのが組み込み開発の部署でした。
マイコンや周辺機器と向き合いながら、**ハードウェアに近いレイヤー**でのプログラミングを学びました。

「メモリはギリギリまで使わない」「割り込み処理は最短で抜ける」といった、リソース制約のある世界での考え方は、今でも自分のコードの書き方に染み込んでいます。

### Kotlin との出会い → Android 開発へ

組み込みをやっていた頃、業務の合間に Kotlin を触る機会がありました。
その表現力と書き心地に一気に引き込まれ、**Android アプリ開発**の世界へ転向。

ネイティブアプリ特有のライフサイクル管理や非同期処理、UX の細かい作り込みを経験しました。
その後 **iOS 開発**も担当し、Swift で同様の機能を実装するなかでプラットフォームの違いと共通点を肌で学びました。

## 今作っているもの：IssueDock

**IssueDock** は、GitHub の Projects / Issue / Branch / PR を一画面で管理できるツールです。

GitHub の公式 UI だと Projects・Issue・Branch・PR がそれぞれ別の画面に散らばっていて、開発フローを俯瞰しにくいと感じていました。それを解消したくて作り始めました。

### 技術スタック

| 領域 | 技術 |
|---|---|
| バックエンド | Go（自作 REST API） |
| フロントエンド | Nuxt.js（Vue 3） |
| API連携 | GitHub GraphQL API / REST API |

### 直近で実装したこと

**バックエンド（Go）**
- GitHub Projects v2 / Issue CRUD / Branch・PR 紐付け API
- リポジトリ管理 API（プロジェクトへのリポジトリ紐付け）
- Git Flow データ API（ブランチ・コミット履歴を優先度付きで返す）
- ラベル CRUD API、PR 一覧 API

**フロントエンド（Nuxt.js）**
- Git Flow のコミットグラフをベジェ曲線で描画
- Issue 作成モーダル（ラベル選択式、リポジトリ連動）
- Open/Closed/All タブ・リポジトリ絞り込み・キーワード検索

作りながら N+1 問題、ループ内 `defer` のリソースリーク、URL エンコードの使い分けなど、細かいところでよく詰まっています。その気づきをこの note に記録しています。
